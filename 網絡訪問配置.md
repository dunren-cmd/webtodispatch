## 🔧 配置說明

### Vite 配置
應用程式已配置為監聽所有網絡接口：

```typescript
// vite.config.ts
server: {
  port: 3050,
  host: '0.0.0.0',  // 監聽所有接口
  open: true
}
```

### 啟動命令
使用 `--host` 參數啟動，允許網絡訪問：
```bash
npm run dev  # 內部使用 vite --host
```

## 🔥 防火牆設置

如果需要允許外部訪問，請確保防火牆允許以下端口：

### Ubuntu/Debian (ufw)
```bash
# 允許前端應用端口
sudo ufw allow 3050/tcp

# 允許 Supabase 端口（如果需要外部訪問）
sudo ufw allow 54321/tcp
sudo ufw allow 54323/tcp

# 查看防火牆狀態
sudo ufw status
```

### 其他防火牆工具
如果使用其他防火牆工具（如 firewalld, iptables），請相應地開放端口。

## 🔍 檢查網絡配置

### 查看本機 IP
```bash
hostname -I
# 或
ip addr show | grep "inet "
```

### 檢查端口監聽
```bash
netstat -tlnp | grep 3050
# 或
ss -tlnp | grep 3050
```

應該看到類似：
```
*:3050  # 表示監聽所有接口
```

### 測試連接
```bash
# 從本機測試
curl http://192.168.62.101:3050

# 從其他設備測試（替換為實際 IP）
curl http://192.168.62.101:3050
```

## 🛠️ 故障排除

### 無法通過 IP 訪問

1. **檢查 IP 地址**
   ```bash
   hostname -I
   ```

2. **檢查端口監聽**
   ```bash
   netstat -tlnp | grep 3050
   ```
   應該看到 `*:3050` 或 `0.0.0.0:3050`

3. **檢查防火牆**
   ```bash
   sudo ufw status
   ```

4. **重啟應用程式**
   ```bash
   # 停止當前進程
   pkill -f "vite --host"
   
   # 重新啟動
   npm run dev
   ```

5. **檢查網絡連接**
   - 確保設備在同一網絡
   - 嘗試 ping 服務器 IP

## 📝 更新 IP 地址

如果 IP 地址改變，可以通過以下方式查看：

```bash
hostname -I | awk '{print $1}'
```

然後更新訪問地址。
