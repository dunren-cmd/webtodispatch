# 替代方案：手動操作步驟

如果 SQL Editor 出現 API 錯誤，可以使用 Table Editor 手動操作：

## 📋 完整手動操作流程

### 步驟 1：使用 Table Editor 添加 ID 欄位

1. 開啟 Supabase Dashboard：`http://192.168.68.75:54323`
2. 進入 **Table Editor**
3. 選擇 **PersonnelData** 表
4. 點擊 **Add column** 或 **新增欄位**
5. 設定：
   - **Name**: `id`
   - **Type**: 選擇 `int8` 或 `bigint`
   - **Nullable**: 暫時勾選（允許 NULL）
6. 點擊 **Save**

### 步驟 2：生成 ID 值

**選項 A：使用 SQL（如果可以使用）**

執行這個簡單的 UPDATE：

```sql
UPDATE "PersonnelData" 
SET id = FLOOR(RANDOM() * 9000000000 + 1000000000)::BIGINT 
WHERE id IS NULL;
```

**選項 B：手動填入**

在 Table Editor 中，為每筆記錄手動填入 ID（10 位數亂數）

### 步驟 3：添加 employee_id 到 users 表

1. 在 Table Editor 中選擇 **users** 表
2. 點擊 **Add column**
3. 設定：
   - **Name**: `employee_id`
   - **Type**: `text`
   - **Nullable**: 勾選
4. 點擊 **Save**

---

## 🔍 檢查表名

如果找不到 PersonnelData 表，先檢查實際表名：

在 SQL Editor 執行（這個查詢應該比較簡單，不會出錯）：

```sql
SELECT table_name FROM information_schema.tables WHERE table_schema = 'public' ORDER BY table_name;
```

根據結果調整後續操作。

---

## 💡 快速測試

先執行這個最簡單的查詢，測試連接是否正常：

```sql
SELECT 1;
```

如果這個都無法執行，可能是：
1. Supabase 服務未正常運行
2. 連接問題
3. 瀏覽器問題（嘗試重新整理或清除快取）
